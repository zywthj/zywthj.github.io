自己总结的15道前端面试题

  • javascript
  • 前端
  • 面试题

注：以下仅为博主自己搜集资料简化整理而来，如有不妥之处请各位大大使劲拍！

1. AMD与CMD的区别与相同点：

相同点：两种规范都是为了模块化JS进行开发。

   区别：采用AMD规范的是RequrieJs，采用CMD规范的是SeaJs，AMD是提前执行，CMD是延迟执行；AMD是依赖前置，CMD是依赖就近。

2. MVC与MVVM：

  • MVC是指数据.视图.控制器，在MVC里，更关注的是Model不变，View里会包含Model，View可以直接访问Model且依赖Model，而Model并不依赖View;

  • MVVM是指视图完全与数据分离，View不能直接访问Model，Model也不能直接访问View，二者通过ViewModel进行沟通，形成双向数据绑定，真正实现了前端与后台分离。

3.闭包：

1).闭包的概念：闭包就是能够读取其他函数内部变量的函数。

举例：

function a(){
    var i = 0;
    function b(){
        alert(++i);
    }
    return b;
}
    var c = a();
    c(); //1

闭包产生的变量不能被垃圾收集器回收，会一直存在于内存中，内耗很大，IE还会导致内存泄露，解决方法是： 在退出函数之前，将不使用的局部变量全部删除（b=null）。

2).js中垃圾回收机制分为两种：

  • 标记清除：垃圾收集器在运行时会给内存中的所有变量加上标记，它会去掉环境中的变量的标记和被环境中变量引用的变量的标记，此后，如果变量再被标记则表示此变量准备被删除。

  • 引用计数：跟踪记录每个值被引用的次数，当声明一个变量并将一个引用值赋给变量时，引用次数+1，相反，当一个变量脱离该值的引用时，引用次数-1，为0时等待垃圾收集器回收。

4.面向对象如何实现继承：

1）构造继承

2）原型继承

3）call.apply继承

例（构造继承.原型继承）:

function personName(){
     this.name = "zyw"; 

}       
function personAge(){
    this.age = 18;
}
personAge.prototype = new personName();
var age = new personAge();
console.log(age.name); //zyw

6.jsonp的实现，有哪些可以用jsonp的标签？

在js中，直接用XMLHttpRequest请求不同域的数据是不可以的，但是可以引入不同域的js脚本文件，jsonp就是利用这个原理实现的。

带有src属性的标签都可以用实现跨域，比如<script>、<img>、<iframe>等。

7.cookie和localstorage

  • cookie：cookie默认是临时存储的，当浏览器关闭进程的时候自动销毁

  • localstorage: 本地存储, 即使关闭浏览器数据也不会消失.

8.请求头里有哪些东西？post和get的不同？

  • Accept：浏览器能够处理的内容类型

  • Accept-Charset：字符集

  • Accept-Encoding：压缩编码

  • Accept-Language：语言

  • Connection：浏览器与服务器之间连接的类型

  • Cookie：当前页面设置的任何cookie

  • Host：发出请求的页面所在的域

  • Referer：发出请求的页面的URI

  • User-Agent：浏览器的用户代理字符串

    Get常用于向服务器查询某些信息，Post通常用于向服务器发送应该被保存的数据。

9.事件冒泡与捕获，事件委托，target与currenttarget？

1）事件冒泡即事件开始时由具体的元素向上传播到不具体的节点，在事件捕获过程中，document对象首先接收到发生的事件，然后事件沿DOM树向下，一直传播到实际目标。

2）事件委托利用了事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。

3）target是事件目标元素，currenttarget是处理事件的那个元素，始终与对象this相等，例：

document.body.click = function(event){
    alert(event.currenttarget === document.body) //true
    alert(this === document.body) //true
    alert(event.target === document.getElementById("btn")) //true
}

10.前端工程化理解

随着项目复杂度的增加，很多大型应用不再适用于使用传统的模式进行开发，前端工程化就是利用一些工具去提高开发效率，更好地构建和管理代码，比如模块化和组件化.代码版本管理SVN.git，还有 grunt.gulp.fis这些前端自动化工具可以对文件进行压缩合并.图片优化.自动格式化等，都极大的提高了开发效率并且使大型应用中的代码更便于维护和管理。

11.MongoDB与Mysql的区别

存储多种数据类型，有大量的键值对数据需要存储就用mongodb；

事务处理一般用mysql，但是性能比mongodb差点。

12.ES2015的相关特性：

用let来声明变量，用const来声明常量，增加了块级作用域;

支持js模块化，可以用import引入，export导出js文件;

新增了许多语法糖，比如箭头函数，类，解构赋值等。

   14_1.es6 的 import 和 CMD 风格的 require 有什么区别？

   CMD的require是加载时执行而es6的import是调用时执行，加载时只产生引用 ；require可以动态加载，而import只能静态加载。

   14_2.es6 的类:

   es6中引用了class关键字，添加了对类的支持，不过这个类实际上只是对原型进行包装的语法糖。

   14_3.es6 的 Promise 对象的了解:

   一个Promise对象可以理解为一次将要执行的操作（常常被用于异步操作），使用了 Promise 对象之后可以用一种链式调用的方式来组织代码，让代码更加直观。而且由于 Promise.all 这样的方法存在，可以让同时执行多个操作变得简单。

   Promise 对象有三种状态：

      • Fulfilled 可以理解为成功的状态

      • Rejected 可以理解为失败的状态

      • Pending 既不是 Fulfilld 也不是 Rejected 的状态，可以理解为 Promise 对象实例创建时候的初始状态

13.javascript 的哪些东西分别存在于内存的堆和栈 ?

原始数据类型诸如null，undefined，Boolean，String，Number存在于栈中,

引用类型诸如Obeject，Array，function存在于堆中。

15.css媒体查询要注意哪些问题？

1）使用媒体类型：

<link rel="stylesheet" type="text/css" href="site.css" media="screen" />
<link rel="stylesheet" type="text/css" href="print.css" media="print" />

• screen 适用于计算机彩色屏幕。

• print 适用于打印预览模式下查看的内容或者打印机打印的内容。

2） 媒体查询规则：

@media all and (min-width: 800px) { ... }

• @media all 是媒体类型，也就是说，将此 CSS 应用于所有媒体类型。

• (min-width: 800px) 是包含媒体查询的表达式，如果浏览器的最小宽度为 800 像素，则会告诉浏览器只运用对应的 CSS。
